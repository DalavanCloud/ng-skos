ng-skos - API Reference

Welcome to angular-skos API reference.

ng-skos is an AngularJS module to work with terminologies, based on the Simple Knowledge Organization System (SKOS). In short, the module supports concepts (based on SKOS Concept), terminologies (based on SKOS Concept Schemes), and mappings (based on SKOS mapping properties). Concept collections for grouping and sorting concepts are not supported yet.

See http://www.w3.org/2004/02/skos/ for more information about SKOS.

Concepts

A concept in ng-skos is a JSON object with the following keys:

uri
URI of the concept, given as string
prefLabel
object that maps language tags to preferred labels, each given as string
altLabel
object that maps language tags to unordered arrays of alternative labels, each given as string
notation
partly ordered array of notations, each given as string. The first notation might be used as primary notation but multiple notations might also be used equivalently. The data type is assumed to be undefined or implicitly known. To support multiple notations of different type, additional concept keys might be introduced in a later version of ng-skos.

Terminologies

A terminology in ng-skos is a JSON object with the following keys: *

uri
URI of the terminology

an object that maps language tags to preferred names of the terminology, each given as string.

...

topConcepts
unordered array of strings with concept URIs and/or concept objects.

Mappings

A mapping in ng-skos is a JSON object with keys yet-to-be defined.

All keys are optional, but an empty object should better be replaced by

null. Missing keys may indicate that the corresponding properties is not known (open world). To explicitly state for instance that a concept has no notation (closed world), set it no null or to an empty array.

See also ng-skos.skosAccess to look up concepts and terminologies by URI
